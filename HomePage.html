AJAX significa JavaScript asíncrono y XML (Asynchronous JavaScript and XML). Es un conjunto de técnicas de desarrollo web que permiten que las aplicaciones web funcionen de forma asíncrona, procesando cualquier solicitud al servidor en segundo plano. Contar con protocolos específicos para el intercambio de datos es fundamental para que se pueda producir el proceso con todas las garantías de éxito, sea cual sea el lenguaje de programación en el que se base el software empleado o el sistema operativo que use cada ordenador para funcionar. XML y JSON son dos de los más conocidos. Por lo tanto, es indispensable que conozcas sus principales diferencias de cara a elegir el más adecuado en cada caso.

XML y JSON

El formato JSON se utiliza para estructurar datos en forma de texto y permite el intercambio de información entre aplicaciones de manera sencilla, liviana y rápida. Por eso, es tan importante saber cómo se estructura y las principales diferencias con el modelo XML.

AJAX

Ajax permite que un usuario de la aplicación web interactúe con una página web sin la interrupción que implica volver a cargar la página web. La interacción del sitio web ocurre rápidamente sólo con partes de la página de recarga y renovación.
